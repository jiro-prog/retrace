# 03. 類似プロジェクト調査レポート

**調査日**：2026年4月17日
**目的**：Retraceのコンセプトと被る先行プロジェクトの洗い出し

## 結論

**Retraceのコンセプト独自性は十分高い。**

「スマホ起点＋ローカルLLM推理＋探偵ロールプレイ＋家庭内失くし物×蓄積学習」という組み合わせを満たすプロダクトは、徹底調査の結果見つからなかった。

## カテゴリ別の既存プロダクト

### カテゴリ1：公共落とし物マッチング系（コンセプト別物）
- **落とし物クラウドfind**（JR東日本採用）
- **L&F - Lost & Found App**
- **lofoBot**（2017）
- **Lost & Found AI**

→ 全て「他人が拾った物を本人に繋ぐ」タイプ。自宅内失くし物は対象外。

### カテゴリ2：ChatGPT GPTs系（思想は近い、実装は貧弱）
- **Treasure-Finder Extraordinaire**（最も近い競合、chats数60+で低迷）
- **Home Finder AI**
- **DetectiveGPT**（盗難品探索、家庭内とは違う）

→ コンセプト近似はあるがプロダクトとして成立していない。スタンドアロンアプリでもなく、ローカルLLM/スマホ/蓄積学習は皆無。

### カテゴリ3：ハードウェア/ロボット系（方向性が違う）
- **MIT RFusion**（2021） — ロボットアームで物理探索
- **IEEE論文のLLMロボット**（2025） — LLMで家の変化を理解
- **Microsoft Find My Things** — 視覚障害者向けカメラ認識

→ ハードウェア前提。ソフト単体の推理アプリとは土俵が違う。

### カテゴリ4：位置リマインダー系（別ジャンル）
- Google Keep、RemindMeWhere、Pinky Map、Any.do等

→ ユーザーが能動的に場所登録するタイプ。推理で探す発想ではない。

### カテゴリ5：AI推理ゲーム系（用途が違う）
- **みんなで推理**（Steam/スマホ）
- **未解決**（iOS）

→ エンタメゲーム。実用の失くし物探しではない。

## Retraceの独自性ポジション

以下の5軸の組み合わせで既存と差別化できる：

1. **自宅内の個人失くし物特化**（公共落とし物DBとは別市場）
2. **スマホ起点のパニック時UX**（デスクトップGPTsでは代替不可）
3. **ローカルLLM×プライバシー完結**（クラウドAI系との明確な差別化）
4. **探偵ロールプレイによるゲーム感**（続く要因）
5. **蓄積による個人パターン学習**（GPTs系には原理的に実装困難）

特にローカルLLM×自宅Wi-Fi×スマホ連携という技術的ポジションは、調査範囲では**完全に空白地帯**。

## 将来の脅威

以下は今後注視すべき動き：

- **IEEE論文のLLMロボット研究**が製品化に進む可能性（当面は別市場）
- **ChatGPTアプリの機能拡張**（音声・蓄積対応でカジュアル層が奪われるリスク）
- **Apple Intelligence / Gemini on Android**が「物の場所を覚える」機能を標準実装する可能性（数年スパン）

いずれも**プライバシー完結**や**探偵ロールプレイ**を差別化軸として残すことで対応可能。

## Treasure-Finder Extraordinaire（chats数60+で低迷）から学ぶこと

「AIに質問して失くし物を探す」コンセプト自体は既出だが、誰も使っていない。

**なぜか？ → デスクトップでChatGPTを開く動線が失くし物シーンに合っていない。**

Retraceの「スマホ起点じゃないと成立しない」という設計判断は、既存GPTs系が失敗した理由を本質的に突いている。

## 公開時に意識すべきメッセージ

紹介文・README・note記事で以下を明記しておく：

1. 「ChatGPTのお探しGPTとは違う、スマホで完結する助手体験」
2. 「プライバシーを守って、自分の家のLLMに相談する」
3. 「使うほど探偵が成長する（GPTsには真似できない蓄積学習）」

これらを最初から明示することで、後から「類似あるじゃん」と言われた時の反論が効く。
